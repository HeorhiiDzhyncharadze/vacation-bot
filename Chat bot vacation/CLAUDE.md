# Vacation Calculator — контекст проєкту

Проєкт для розрахунку доступної відпустки за строковим контрактом.
Дві реалізації спільної логіки: Excel-калькулятор і Telegram-бот.

## Логіка розрахунку (спільна, критична)

Відпустка нараховується **лінійно й рівномірно** по місяцях контракту.

- `duration = (end_month - start_month) % 12 + 1` — тривалість у місяцях, включно; працює через Новий рік (напр. листопад→березень = 5).
- `rate = total_hours / duration` — нарахування за місяць.
- Для кожного місяця `n` (1..duration) наростаючим підсумком:
  - `accrued = rate * n`
  - `available_hours = max(0, accrued - used_hours)`  ← від'ємне завжди затискається в 0
  - `available_days = available_hours / shift_hours`
- Підсумок: `remaining = max(0, total - used)`, у днях `/ shift_hours`.

Еталонний тест-кейс (має давати саме це):
`total=168, used=72, start=Січень(1), end=Жовтень(10), shift=12`
→ duration=10, rate=16.8; Травень=12.0год/1.0дн; Червень=28.8/2.4; Жовтень=96.0/8.0;
січень–квітень = 0 (бо used ще перевищує накопичене).

## Файли

- `vacation_bot.py` — Telegram-бот (python-telegram-bot >= 21, long-polling, ConversationHandler).
  Питає по черзі: усього годин → використано → місяць початку (кнопки) → місяць кінця (кнопки) → годин у зміні. Видає таблицю по місяцях.
  Токен задається в константі `TOKEN`.
- `requirements.txt` — залежності.
- `vacation_calculator.xlsx` — той самий розрахунок у Excel: named ranges, dropdown місяців, дата-бари, автопідсвітка поточного місяця, clamp у 0.

## Стек

- Python 3.11+
- python-telegram-bot >= 21 (async)
- openpyxl (для xlsx, якщо генеруватимемо файл з коду)
